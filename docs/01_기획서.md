## 정보 신뢰도 표기 원칙

이 기획서는 세 가지 신뢰도 수준으로 정보를 구분한다:

| 표기 | 의미 |
|------|------|
| ✅ 확인됨 | 공식 문서·연구 에이전트 조사로 확인된 사실 |
| ⚠️ 확인 필요 | 개발 착수 전 직접 확인이 필요한 항목 |
| *(표기 없음)* | 설계 의도·방향성이며 정확한 수치는 구현 시 결정 |

**개발 시작 전 반드시 직접 확인해야 할 항목:**
1. 각 언론사 RSS 피드 실제 URL 및 구조 (robots.txt 포함)
2. 공공데이터포털 데이터셋별 정확한 API 엔드포인트 및 갱신 주기
3. 정부24 API 제공 범위 (다문화 지원사업 포함 여부)
4. 이주민센터친구 공식 웹사이트 URL
5. 최신 다문화가족 현황 통계 수치 및 출처

## 1. 프로젝트 배경 및 목표

### 배경
- 국내 다문화 가족 수는 지속 증가 추세 (정확한 최신 수치는 행정안전부 외국인주민현황 통계에서 확인 필요)
- 관련 뉴스·법령·지원사업 정보가 **여러 사이트에 분산** 되어 있어 접근성 낮음
- 현장 실무자(사회복지사, 상담사)는 매번 수십 개 사이트를 순회해야 함
- 정책 변화가 빠른 분야임에도 **실시간 통합 정보 허브** 부재

### 프로그램 목표
| 구분 | 목표 |
|------|------|
| 핵심 가치 | 다문화 관련 정보를 **한 곳에서, 빠르게, 이해하기 쉽게** 제공 |
| 일차 목표 | 뉴스·법령·지원사업 카테고리별 통합 뷰어 |
| 이차 목표 | 자동 업데이트·알림으로 최신 정보 유지 |
| 삼차 목표 | 다국어 지원으로 당사자도 직접 사용 가능 |

## 2. 개발 방법론 선택

### PDCA 대신 추천: **Design Thinking × OKR × Agile Sprint**

#### 왜 이 조합인가?

```
[Design Thinking]         [OKR]                  [Agile Sprint]
사용자 공감 → 문제 정의   목표·지표 설정          2주 단위 반복 개발
프로토타이핑 → 테스트     달성 여부 측정           빠른 피드백 반영
```

| 방법론 | 역할 | 적용 시점 |
|--------|------|-----------|
| **Design Thinking** | "무엇을 만들까" 결정 | 기획 단계 |
| **OKR** | "성공 기준" 설정 | 전 과정 |
| **Agile Sprint** | "어떻게 만들까" 실행 | 개발 단계 |

#### Design Thinking 5단계 적용
```
1. Empathize  → 사용자 인터뷰 (사회복지사 3명, 다문화 당사자 2명)
2. Define     → "○○ 사용자는 □□ 상황에서 △△ 정보가 필요하다"
3. Ideate     → 기능 아이디어 브레인스토밍
4. Prototype  → 와이어프레임 → Figma 목업
5. Test       → 사용성 테스트 → 반영
```

#### PDCA보다 이 방법론이 나은 이유
- PDCA: **결과 개선** 중심 (이미 만든 것을 반복 개선)
- Design Thinking + OKR + Agile: **사용자 문제** 에서 출발, 목표 지표로 성공 측정, 빠른 반복

## 4. 경쟁/참고 서비스 분석

### 주요 참고 서비스

| 서비스 | URL | 강점 | 약점 |
|--------|-----|------|------|
| **다누리** | danuri.go.kr | 13개국어 지원, 결혼이민자 특화 | API 없음, RSS 불안정, UI 노후화 |
| **하이코리아** | hikorea.go.kr | 체류·비자 행정 처리 강점 | 4개 언어, 구조 복잡, 비전문가 이해 어려움 |
| **법제처** | law.go.kr | 공식 Open API 제공, 개정이력 완전 | 법령 문어체, 생활 연결 없음 |
| **공공데이터포털** | data.go.kr | 다수 데이터셋 API 무료 | 원시 데이터, 갱신 불규칙 |
| **정부24** | gov.kr | 지원사업 통합 | 다문화 특화 아님 |
| **이주민센터친구** | migrant.or.kr *(URL 개발 전 재확인 필요)* | 현장·인권 이슈 강함 | API/RSS 없음, 업데이트 불규칙 |

### ⭐ 결정적 발견: Windows 데스크탑 앱 **완전 공백**

> 조사 결과, 한국 다문화 정보를 위한 **Windows 데스크탑 전용 애플리케이션은 단 하나도 존재하지 않는다.**
> 모든 기존 서비스는 웹 브라우저 기반이며, 일부만 모바일 앱(Android/iOS)을 제공한다.
> 이 프로그램은 **블루오션**이다.

### 갭 분석 (우리 프로그램이 채울 공간)

```
[기존 서비스의 공통 문제점]
├── 정보 분산: 사용자가 여러 사이트를 전전함
├── 검색 취약: 키워드 매칭 수준, 의미 검색 없음

[우리 프로그램의 차별점]
├── 통합 뷰어: 단일 앱에서 전체 카테고리 접근
├── 스마트 필터: 지역 × 국적 × 대상 × 날짜 복합 필터

├── AI 요약: 복잡한 법령을 쉬운 말로 요약
├── 로컬 캐시: 오프라인에서도 최근 데이터 열람
└── 개인 북마크: 자주 쓰는 정보 즐겨찾기
```

## 5. 기능 요구사항

### 5.1 핵심 기능 (MVP - Must Have)

#### M1. 뉴스 뷰어
- RSS 피드 기반 다문화 뉴스 자동 수집 (연합뉴스, KBS 등 RSS 제공 언론사 대상 — 실제 지원 언론사 수는 개발 전 RSS 수집 조사 후 확정)
- 카테고리: 정책/제도, 지역소식, 국제, 인터뷰/사례
- 키워드 필터, 날짜 필터, 언론사 필터
- 기사 미리보기 + 원문 링크

#### M2. 법령 브라우저
- 법제처 API 연동 (다문화가족지원법, 재한외국인처우기본법, 출입국관리법 등)
- 조문별 검색
- 개정 이력 표시 (최신 vs 이전)
- 조문 북마크 및 메모 기능

#### M3. 지원사업 데이터베이스
- 중앙정부 + 지자체 지원사업 목록
- 필터: 지역, 대상(국적/연령/체류자격), 분야(교육/의료/주거/취업)
- 신청 기간, 지원 내용, 담당 기관 표시
- 마감 임박 알림

#### M4. 검색 엔진
- 전체 카테고리 통합 검색
- 한국어 형태소 분석 기반 (KoNLPy)
- 자동완성, 최근 검색어, 인기 검색어

#### M5. 업데이트 스케줄러
- 백그라운드 자동 수집 (기본: 매 1시간)
- 마지막 업데이트 시간 표시
- 수동 새로고침 버튼

### 5.2 확장 기능 (Should Have)

#### S1. 알림 시스템
- 법령 개정 알림 (Windows 토스트 알림)
- 관심 키워드 등록 → 관련 뉴스 알림
- 지원사업 마감 D-7, D-1 알림

#### S2. 다국어 지원
- UI 언어: 한국어, 영어, 중국어(간체), 베트남어, 태국어, 필리핀어
- 기사 자동 번역 (DeepL API or LibreTranslate 오픈소스)
- 법령 요약 → 다국어 요약 생성

#### S3. AI 요약/해설
- Claude API or Ollama(로컬 LLM)로 법령 쉬운 말 요약
- "이 법은 나에게 해당되나요?" 판단 도우미
- 뉴스 3줄 요약

#### S4. 내보내기
- 검색 결과 Excel/CSV 저장
- 법령 조문 PDF 저장
- 뉴스 클리핑 Word 문서 저장

#### S5. 개인화 대시보드
- 자주 보는 카테고리 즐겨찾기
- 최근 열람 이력
- 맞춤 뉴스피드 설정

### 5.3 미래 기능 (Nice to Have)

#### N1. 통계/시각화
- 다문화 관련 뉴스 트렌드 차트
- 법령 개정 타임라인
- 지원사업 지역별 분포 지도

#### N2. 커뮤니티 연동
- 다문화 관련 카페/커뮤니티 최신 글 수집

#### N3. 캘린더 연동
- 지원사업 신청 기간을 Windows 캘린더에 추가

## 7. UI/UX 설계 원칙

### 디자인 철학: "신뢰할 수 있는 공공 정보 도구"

```
원칙 1: 명확성 (Clarity)
- 출처·날짜 항상 표시
- 공식 정보 / 언론 정보 / AI 요약 구분 표시

원칙 2: 접근성 (Accessibility)
- 글자 크기 3단계 조절 (소/중/대)
- 고대비 모드 지원
- 키보드 단독 조작 가능
```

### 9.2 RSS 피드 (뉴스 수집)

> ⚠️ **개발 전 반드시 직접 확인 필요**: 아래 언론사들은 RSS를 제공하나, 정확한 URL 경로와 구조는 해당 언론사 사이트에서 직접 확인해야 한다. URL을 코드에 하드코딩하기 전에 실제 접속 테스트를 거칠 것.

```python
NEWS_SOURCES = [
    # 아래 URL은 예시 구조이며, 개발 전 실제 URL 확인 필수
    {"name": "연합뉴스", "url": "TBD - yna.co.kr 에서 RSS 경로 확인"},
    {"name": "KBS뉴스",  "url": "TBD - news.kbs.co.kr 에서 RSS 경로 확인"},
    {"name": "뉴시스",   "url": "TBD - newsis.com 에서 RSS 경로 확인"},
    # 다문화 전문 미디어 — RSS 제공 여부 사전 조사 필요
    {"name": "이주민방송MWTV", "url": "TBD - 공식 홈페이지에서 확인"},
    # 지역 언론 — 지역별 RSS 제공 현황 조사 후 추가
]
```

**RSS 수집 전 체크리스트**
```
□ 각 언론사 robots.txt 확인 (수집 허용 여부)
□ RSS 피드 실제 URL 접속 테스트
□ 피드 내 필드 구조 파악 (title/link/pubDate/description)
□ 다문화 키워드 필터링 가능 여부 확인
□ 이용 약관에 자동 수집 금지 조항 유무 확인
```

### 9.3 법적 고려사항


## 13. 사용자가 놓칠 수 있는 핵심 고려사항

### ⚠️ 반드시 포함해야 할 항목

#### 1. 저작권 및 법적 이슈
```
- 뉴스 기사 전문은 저작권 보호 대상
  → 제목 + 첫 2문장 + 원문 링크 방식으로 저장
- 공공기관 데이터는 CC-BY, 출처 표기 의무
- 크롤링 전 robots.txt 확인 필수
```

#### 2. 개인정보보호
```
- 프로그램 자체에서 수집하는 사용자 데이터 없음
  → Privacy by Design 원칙 적용
- 로그는 로컬만 저장, 외부 전송 금지
- 설정에서 전체 데이터 초기화 기능 제공
```

#### 3. 다국어 UI vs 다국어 콘텐츠 구분
```
- UI 다국어: 버튼/메뉴 텍스트 번역 (i18n 라이브러리)
- 콘텐츠 다국어: 기사/법령 자동 번역 (API 비용 발생)
  → 콘텐츠 번역은 선택적 기능으로 구현 (비용 절감)
```

#### 4. 정보 최신성 표시 의무
```
- 법령: "시행일자", "마지막 개정일" 항상 표시
- 지원사업: "마감일", "업데이트 일자" 표시
- 자동 수집 실패 시 "X시간 전 데이터" 경고 표시
```

#### 5. 접근성 (Accessibility)
```
- 고령의 결혼이민자, 시각 제한 사용자 고려
- 글자 크기: 3단계 (기본 12pt, 중 15pt, 대 18pt)
- 고대비 모드: 흰 배경/검은 글씨 ↔ 검은 배경/흰 글씨
- 키보드 단독 탐색 지원 (Tab, Enter, 방향키)
```

#### 6. 오프라인 동작
```
- 사회복지사가 현장 방문 시 Wi-Fi 없는 환경
- 마지막 수집 데이터는 항상 로컬 DB에 보관
- 오프라인 상태 명확히 표시 + 최근 데이터로 계속 사용 가능
```

#### 7. 지역별 필터링 (매우 중요!)
```
- 지원사업은 지역마다 달라 전국 일괄 표시는 혼란 초래
- 사용자의 기본 지역 설정 → 관련 지원사업 우선 노출
- 광역시도 + 시군구 2단계 지역 필터
```

#### 8. 데이터 갱신 실패 처리
```
- API 서버 점검/오류 시 프로그램이 멈추지 않아야 함
- 재시도 로직 (3회) + 지수 백오프
- 실패 시 마지막 성공 데이터 표시 + 사용자에게 안내
```

#### 9. 프로그램 자동 업데이트
```
- 기능 추가/버그 수정 시 사용자에게 업데이트 알림
- GitHub Releases 기반 자동 업데이트 체크
```

#### 10. 설치 환경 고려
```
- 일부 기관 PC는 관리자 권한 없음 → Portable 버전도 제공
- Python 런타임 포함 단일 .exe 배포 (PyInstaller)
- 지원 OS: Windows 10, Windows 11 (PyQt6는 Windows 7 공식 미지원)
- QWebEngineView 사용 시 Visual C++ Redistributable 의존성 확인 필요
```
