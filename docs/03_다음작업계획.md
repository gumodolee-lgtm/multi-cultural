## 작업 1: 생활 영역 5개 뷰 구현

다누리 A안 — 생활영역별 메뉴 구조. 각 뷰는 해당 분야의 뉴스 + 법령 + 지원사업을 통합 표시.

### 1-1. 🛂 비자·체류 (visa)
- 관련 법령: 출입국관리법, 국적법
- 관련 지원: KIIP(사회통합프로그램), 체류자격 변경 안내
- 관련 뉴스: 출입국 정책 변경, 비자 제도 개편
- UI: 상단 요약 카드 + 법령/뉴스/지원사업 섹션별 표시

### 1-2. ⚕️ 의료·복지 (health)
- 관련 법령: 다문화가족지원법 (의료 조항)
- 관련 지원: 의료비 지원, 건강검진, 의료 통역
- 관련 뉴스: 의료·복지 정책
- UI: 동일 통합 구조

### 1-3. 👶 가족·육아 (family)
- 관련 법령: 다문화가족지원법 (가족 조항)
- 관련 지원: 방문교육, 이중언어 환경조성, 자녀 교육
- 관련 뉴스: 가족·육아 관련
- UI: 동일 통합 구조

### 1-4. 🎓 교육·문화 (education)
- 관련 법령: 다문화가족지원법 (교육 조항), 재한외국인처우기본법
- 관련 지원: 한국어 교육, 문화 프로그램, 방과후 프로그램
- 관련 뉴스: 교육·문화 정책
- UI: 동일 통합 구조

### 1-5. 💼 일자리 (job)
- 관련 법령: 외국인근로자 고용법
- 관련 지원: 취업 연계, 직업 교육, 자격증 지원
- 관련 뉴스: 고용·노동 정책
- UI: 동일 통합 구조

### 구현 방향
- `LifeAreaView` 공통 베이스 클래스 생성 → 5개 뷰가 상속
- 각 영역별 키워드 매핑으로 Mock 데이터에서 관련 항목 필터링
- 향후 실제 서비스 연동 시 키워드 기반 DB 쿼리로 교체

## 작업 3: 데이터 자동 수집 시스템 (Phase 2 핵심)

사용자 요구: "모든 것이 자동으로 이뤄졌으면 좋겠다"

### 설계 방향

```
프로그램 시작 (main.py)
  │
  ├── 1단계: 즉시 수집 (Startup Fetch)
  │   └── 마지막 수집 시간 확인 → 일정 시간 경과 시 즉시 새 데이터 fetch
  │
  ├── 2단계: 백그라운드 스케줄러 시작 (APScheduler)
  │   ├── 뉴스 RSS 수집: 매 60분 (config.yaml 설정)
  │   ├── 법령 API 확인: 매 24시간
  │   └── 지원사업 확인: 매 12시간
  │
  ├── 3단계: 시스템 트레이 상주 (선택)
  │   ├── 프로그램 닫아도 백그라운드 동작
  │   └── 새 정보 발견 시 Windows 토스트 알림
  │
  └── 4단계: UI 자동 갱신
      └── 새 데이터 수집 완료 → 현재 보고 있는 뷰 자동 리프레시
```

### 구현할 서비스 모듈

| 파일 | 역할 |
|------|------|
| `app/services/news_service.py` | RSS 피드 수집, 키워드 필터링, DB 저장 |
| `app/services/law_service.py` | 법제처 API 연동, 법령 조회/저장 |
| `app/services/support_service.py` | 공공데이터포털 API, 지원사업 조회/저장 |
| `app/services/search_service.py` | 통합 검색 (형태소 분석 포함) |
| `app/scheduler/scheduler.py` | APScheduler 설정, 작업 등록/관리 |
| `app/scheduler/tasks.py` | 수집 작업 함수 (뉴스/법령/지원사업) |

### 수집 주기 (config.yaml에서 변경 가능)

| 데이터 | 기본 주기 | 소스 |
|--------|----------|------|
| 뉴스 | 60분 | RSS 피드 (연합뉴스, KBS 등) |
| 법령 | 24시간 | 법제처 Open API |
| 지원사업 | 12시간 | 공공데이터포털 API |
